# 신뢰성 정책 (Reliability)

## 서비스 수준 목표 (SLO)

| 항목 | 목표 | 측정 방법 |
|---|---|---|
| 빌드 성공률 | 100% (main 브랜치) | CI 게이트 |
| 테스트 통과율 | 100% | `./gradlew test` |
| 재고 동시성 오류 | 0건 | P-1 동시성 테스트 |

## 장애 등급 정의

| 등급 | 조건 | 대응 |
|---|---|---|
| P0 | 재고 음수 발생, 주문 이력 손실 | 즉시 롤백 + 하네스 개선 |
| P1 | N+1 쿼리 발생, API 응답 형식 변경 | 당일 수정 |
| P2 | 로그 누락, 응답 지연 | 다음 스프린트 처리 |

## 에이전트 장애 대응 절차

1. `./gradlew test` 실행 → 실패 테스트 격리
2. `docs/constraints/enforcement-map.md` 확인 → 관련 규칙 상태 점검
3. 즉각 수정 후 전체 게이트 재통과
4. P0 장애 시 `docs/improvements/`에 개선 기록 필수

## 롤백 기준

다음 조건 중 하나라도 해당되면 즉시 롤백한다:
- `./gradlew test` 실패
- 재고가 0 미만으로 감소
- 주문 이력에서 상품 정보 조회 불가
