# 제품 요구사항

## 핵심 사용자 시나리오

| ID | 시나리오 | 우선순위 |
|---|---|---|
| US-1 | 운영자가 상품을 등록·수정·삭제할 수 있다 | P0 |
| US-2 | 고객이 상품을 주문할 수 있다 | P0 |
| US-3 | 주문 시 재고가 자동으로 차감된다 | P0 |
| US-4 | 재고가 부족한 상품은 주문할 수 없다 | P0 |
| US-5 | 삭제된 상품의 주문 이력은 보존된다 | P1 |
| US-6 | 주문 목록을 페이지 단위로 조회할 수 있다 | P1 |

## 비기능 요구사항

| 항목 | 기준 | 강제 수단 |
|---|---|---|
| 동시성 | 동시 주문에서 재고 초과 없음 | ADR-0002, P-1 테스트 |
| 데이터 정합성 | 주문 이력은 상품 삭제 후에도 조회 가능 | ADR-0001 |
| 성능 | 주문 목록 조회: N+1 쿼리 없음 | ADR-0003, O-3 테스트 |

## 범위 외 (Out of Scope)

- 인증·인가
- 결제 처리
- 주문 취소·환불
- 상품 카테고리·검색

범위를 확장할 경우 반드시 이 파일과 `docs/adr/`에 결정을 기록한다.
