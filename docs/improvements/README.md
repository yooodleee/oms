# 하네스 개선 기록 (Harness Improvement Log)

이 디렉토리는 버그 탈출·반복 실수·게이트 무력화 사례와 그에 따른 **하네스 개선 내역**을 기록한다.

코드 수정 기록이 아니다. 하네스(테스트, ArchUnit, ADR, enforcement-map)가 어떻게 강화됐는지를 기록한다.

---

## 파일 명명 규칙

```
YYYY-MM-DD-<이슈-키워드>.md
예: 2026-04-15-soft-delete-query-missing-filter.md
```

## 개선 기록 템플릿

```markdown
# [날짜] 이슈 제목

## 발생한 문제
무엇이 잘못됐는가. 어떻게 발견됐는가 (테스트 실패 / 런타임 / 코드 리뷰).

## 탈출 경로 분석
어떤 하네스 레이어(L1~L5, ARCH-1~3)를 통과했는가.
왜 그 레이어가 잡지 못했는가.
관련 규칙 ID (P-1, O-3 등): 당시 enforcement-map 상태는?

## 즉각 수정
변경된 파일과 수정 내용 요약.

## 하네스 개선
- [ ] 새 테스트 추가: `<테스트 클래스>#<메서드명>`
- [ ] ArchUnit 규칙 추가: `<규칙 설명>`
- [ ] ADR 작성: `docs/adr/NNNN-<제목>.md`
- [ ] domain-rules.md 갱신
- [ ] enforcement-map.md 상태 변경: ❌ MISSING → ✅ ENFORCED

## 재발 방지 확인
개선 후 동일 문제가 발생하면 어느 게이트에서 잡히는가.
```
